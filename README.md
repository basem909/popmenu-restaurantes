# Popmenu Restaurantes API

This project exposes a JSON API for restaurants, menus, menu items, authentication, and asynchronous imports.

## Development Setup

1. Install Ruby 3.2.4 and PostgreSQL.
2. Install Redis (for Sidekiq).
3. Run `bundle install`.
4. Create and migrate the database:
   ```bash
   bin/rails db:create db:migrate
   bin/rails db:seed
   ```
5. Start the Rails server:
   ```bash
   bin/rails server
   ```
6. Start Sidekiq in a separate window to process background jobs:
   ```bash
   bundle exec sidekiq -C config/sidekiq.yml
   ```

Sidekiq uses the `REDIS_URL` environment variable (`redis://127.0.0.1:6379/0` by default). Jobs enqueued by the imports API (`Imports::RestaurantTreeWorker`) will run once Sidekiq is connected.

## Authentication

- Register via `POST /api/v1/users`.
- Sign in via `POST /api/v1/users/sign_in` to receive a JWT token in the response body.
- Pass the token in the `Authorization` header (`Bearer <token>`) for protected endpoints such as `POST /api/v1/imports/restaurants`.

## Importing JSON Menus

- Upload the provided `restaurant_data.json` (or any payload using the same schema) via:
  ```bash
  curl -X POST http://localhost:3000/api/v1/imports/restaurants \
       -H "Authorization: Bearer <token>" \
       -H "Content-Type: application/json" \
       -d @restaurant_data.json
  ```
- Alternatively, run the importer directly:
  ```bash
  bin/rails runner 'Imports::RestaurantTreeImporter.new(JSON.parse(File.read("path/to/restaurant_data.json"))).call'
  ```
- The importer logs a summary (`imports.restaurant_tree.completed`) and, when triggered via the API, enqueues `Imports::RestaurantTreeWorker`, which emails the initiating user once processing finishes.
